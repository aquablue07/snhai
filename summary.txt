\f0\b \cf0 3. Design Decisions\
\pard\pardeftab720\sa240\partightenfactor0

\f1\b0 \cf0 \
Based on the requirements and iterative testing, we made the following key design decisions:\
\pard\tx220\tx720\pardeftab720\li720\fi-720\sa240\partightenfactor0
\ls2\ilvl0
\f0\b \cf0 \kerning1\expnd0\expndtw0 \outl0\strokewidth0 {\listtext	\uc0\u8226 	}\expnd0\expndtw0\kerning0
\outl0\strokewidth0 \strokec2 Hybrid Model Architecture
\f1\b0 : We chose a hybrid system that uses a fine-tuned classifier (
\f2\fs26 FinBERT
\f1\fs24 ) for the decision and a deterministic script for the reasoning. This approach uses the LLM for its strength in pattern recognition while using the script to guarantee that the explanations are 100% accurate and auditable according to the business logic. This provides the flexibility of an AI model with the reliability of a rule-based system.\
\ls2\ilvl0
\f0\b \kerning1\expnd0\expndtw0 \outl0\strokewidth0 {\listtext	\uc0\u8226 	}\expnd0\expndtw0\kerning0
\outl0\strokewidth0 \strokec2 Model Selection (
\f3\fs26 ProsusAI/finbert
\f0\fs24 )
\f1\b0 : We selected a BERT-based model pre-trained on a large corpus of financial text. As the task involves specific financial terminology, a domain-specific model like FinBERT has a significant "head start" and can learn the classification task more efficiently from a smaller dataset than a general-purpose model.\
\ls2\ilvl0
\f0\b \kerning1\expnd0\expndtw0 \outl0\strokewidth0 {\listtext	\uc0\u8226 	}\expnd0\expndtw0\kerning0
\outl0\strokewidth0 \strokec2 Parameter-Efficient Fine-Tuning (LoRA)
\f1\b0 : We chose to implement LoRA for the training process. LoRA is vastly more efficient in terms of computational resources (VRAM, time) and storage, as it only trains a small fraction of the model's parameters (less than 1%). It provides performance comparable to a full fine-tune, making the solution more practical and scalable.\
\ls2\ilvl0
\f0\b \kerning1\expnd0\expndtw0 \outl0\strokewidth0 {\listtext	\uc0\u8226 	}\expnd0\expndtw0\kerning0
\outl0\strokewidth0 \strokec2 Synthetic Data Generation
\f1\b0 : We developed a script to generate a balanced dataset from the rules. This process includes creating specific "hard cases" to teach the model about rule precedence and using descriptive reasons for 
\f2\fs26 APPROVE
\f1\fs24  outcomes to reduce model bias. This ensures the model is trained on controlled, high-quality data that covers all decision types.\
\pard\pardeftab720\sa240\partightenfactor0
\cf0 \
\pard\pardeftab720\sa319\partightenfactor0
